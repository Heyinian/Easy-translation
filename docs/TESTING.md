## 1. 测试前准备

建议先确认以下条件：

1. 使用 Python 3.12 环境
2. 使用 .venv312 启动项目
3. 已安装 requirements.txt 中的依赖
4. 如需测试截图 OCR，已安装 Tesseract

推荐启动命令：

```powershell
.\.venv312\Scripts\python.exe app.py
```

自动化 smoke baseline 命令：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_smoke_tests.ps1
```

当前自动化覆盖范围：

1. `app.py` 的入口路径注入
2. `config.py` / `settings_manager.py` 的基础加载
3. runtime 目录覆盖与默认文件初始化
4. `TranslatorCore` 的基础无网络行为

当前自动化**不覆盖**：

1. PyQt6 主窗口真实显示与托盘交互
2. 全局热键真实监听
3. 剪贴板与外部输入框的端到端交互
4. OCR 和打包产物验证

## 2. 冒烟测试

每次改动后，建议至少完成以下冒烟测试：

1. 应用可以正常启动
2. 主窗口可见或可从托盘恢复
3. 输入框手动翻译可用
4. 剪贴板翻译可用
5. 输入窗口翻译可触发
6. 程序未出现明显崩溃或卡死
7. 关闭或最小化后应用仍可从托盘恢复
8. 重复启动时不会出现多个实例

## 3. 功能测试清单

### 3.1 主窗口手动翻译

步骤：

1. 在主输入框输入英文或中文文本
2. 点击“翻译”按钮
3. 检查输出框是否出现译文

通过标准：

1. 不报错
2. 输出框出现合理译文

### 3.2 剪贴板翻译

步骤：

1. 复制一段文本到系统剪贴板
2. 点击“由剪贴板翻译”按钮，或使用剪贴板热键
3. 检查主窗口输入框和输出框是否更新

通过标准：

1. 输入框显示剪贴板原文
2. 输出框显示译文

### 3.3 输入窗口翻译：triple_space 模式

前提：

1. 设置中的输入窗口翻译触发键为 triple_space
2. 三击空格窗口期设置为 1.2 秒或更宽松

步骤：

1. 打开记事本或其他标准文本输入框
2. 输入一段文本
3. 在窗口期内连续按 3 次空格

通过标准：

1. 程序触发翻译
2. 外部输入框中的原文被译文替换
3. 程序不崩溃
4. 触发过程中不会唤醒主窗口
5. 点按较慢或抓取失败时，不会预先删除外部输入框中的原文
6. 触发后继续输入空格时，不会因为残留全选状态而覆盖整段文本

### 3.4 输入窗口翻译：组合键模式

前提：

1. 设置中的输入窗口翻译触发键为组合键，例如 alt+shift+x

步骤：

1. 打开记事本或其他标准文本输入框
2. 输入一段文本
3. 触发设置中的组合键

通过标准：

1. 程序触发翻译
2. 外部输入框中的原文被译文替换

### 3.5 截图 OCR 翻译

前提：

1. 已安装 Tesseract
2. 设置页中的 Tesseract 路径有效

步骤：

1. 点击截图翻译按钮或使用截图热键
2. 选择包含文本的区域
3. 检查 OCR 结果是否进入主输入框并自动翻译

通过标准：

1. 能完成截图
2. 能识别出可读文本
3. 能自动进入翻译流程

### 3.6 未安装 Tesseract 时的设置保存

前提：

1. 本机未安装 Tesseract，或设置页中的 Tesseract 路径留空

步骤：

1. 打开设置页
2. 修改任意与 OCR 无关的设置项，例如热键或默认翻译源
3. 保存设置

通过标准：

1. 设置保存成功
2. 不因 Tesseract 缺失而阻塞保存
3. 普通翻译功能仍可使用

### 3.7 单实例保护

步骤：

1. 启动应用并保持运行
2. 再次双击启动应用或再次执行启动命令
3. 观察是否出现第二个独立实例

通过标准：

1. 不会保留两个同类实例
2. 已运行实例会被唤醒或显示到前台
3. 新实例会快速退出

## 4. 回归重点

以下区域在近期改动较多，每次改动后建议重点回归：

1. main_window.py 中的翻译线程生命周期
2. 输入窗口翻译触发模式切换
3. 外部输入框 Ctrl+A / Ctrl+C / Ctrl+V 流程
4. 设置页保存后运行时配置重载
5. translate_result() 的单结果返回结构兼容性
6. Tesseract 缺失时的设置保存与 OCR 降级行为
7. 单实例保护与唤醒主窗口行为

## 5. 常见失败场景

### 5.1 三击空格无反应

优先检查：

1. 当前触发模式是否真的是 triple_space
2. 窗口期是否太短
3. 目标程序是否拦截按键
4. 检查 `logs/triple_space.log` 中是否出现 click_count 累计到 3

### 5.2 组合键无反应

优先检查：

1. 当前触发模式是否已经切换为组合键
2. 热键是否与其他程序冲突
3. 是否忘记保存设置

### 5.3 外部输入框没有被替换

优先检查：

1. 目标输入控件是否支持 Ctrl+A / Ctrl+C / Ctrl+V
2. 是否存在剪贴板权限或安全策略限制
3. 检查 `logs/triple_space.log` 是否出现“Primary capture attempt returned empty content”或“Retry capture still empty”
4. 确认失败时没有出现主窗口误唤醒或外部输入框内容被预先删除

### 5.4 截图 OCR 失败

优先检查：

1. Tesseract 是否安装
2. 路径是否正确
3. 截图区域内文字是否足够清晰

## 6. 建议测试环境

建议优先用以下目标程序测试输入窗口翻译：

1. 记事本
2. VS Code 普通文本编辑区
3. 浏览器中的标准文本输入框

不建议一开始就使用强定制输入控件的软件做首轮验证。
